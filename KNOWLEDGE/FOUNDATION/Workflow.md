# Workflow

## Purpose

Development workflow for OreLabs.

## Pipeline

1. **Repository Synchronization** — Read documentation
2. **Task Analysis** — Understand requirements
3. **Implementation** — Write code following standards
4. **Self Review** — Verify against checklist
5. **Documentation Review** — Update docs if needed
6. **Final Report** — Report completion

## Definition of Done

A task is complete only when:

- Implementation is complete
- Code follows project standards
- Architecture remains consistent
- Documentation is updated
- Self-review passes
