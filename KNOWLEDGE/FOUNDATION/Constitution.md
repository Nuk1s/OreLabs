# Constitution

## Purpose

Core principles that define OreLabs.

## Engineering Overview

OreLabs is built on principles that rarely change. These principles guide every engineering decision.

## Principles

1. **Modular Architecture** — Systems are independent
2. **Single Responsibility** — One system = one job
3. **Data-Driven Design** — Configuration over hardcoding
4. **Expandability** — New features extend, not replace
5. **Maintainability** — Code for long-term
6. **Predictable Communication** — Systems talk through interfaces
7. **Backward Compatibility** — Save files survive updates

## Design Rules

- One Player = One Profile
- Every system owns its own data
- Shared functionality belongs in Services
- Systems communicate through interfaces
- New features extend the architecture
