# Coding Standards

## Purpose

How to write code in OreLabs.

## Naming Rules

- **PascalCase** — Modules, Services, Controllers, Folders
- **camelCase** — Local variables, parameters
- **UPPER_SNAKE_CASE** — Constants

## File Extensions

- `.lua` — ModuleScript
- `.server.lua` — Script
- `.client.lua` — LocalScript

**NEVER use `.module.lua`**

## General Principles

- One responsibility per script
- Avoid unnecessary complexity
- Prefer explicit logic
- Write for long-term maintenance
