# OreLabs AI Context

## Purpose

This document provides the minimum required project context for any AI system working inside OreLabs Studio.

The goal is to allow an AI to understand the current state of the project in a few minutes before beginning work.

This document SHALL describe the current state of the project.

This document SHALL NOT describe future ideas as completed work.

# Working Principles

The AI SHALL:

- Follow approved documentation.
- Respect document hierarchy.
- Read only required documentation.
- Preserve architecture.
- Produce deterministic results.
- Request clarification when necessary.

The AI SHALL NOT:

- Invent project facts.
- Assume undocumented architecture.
- Ignore approved specifications.
- Bypass the Workflow.

# Expected AI Behaviour

When receiving a task, AI SHALL:

1. Identify its role.
2. Identify the task type.
3. Read required documents.
4. Build context.
5. Execute according to Workflow.
6. Validate results.
7. Deliver structured outputs.

---

# Token Efficiency

AI SHALL minimize unnecessary context.

Read only the documentation required for the current task.

Avoid loading unrelated systems.

Prefer focused context over complete repository scans.
