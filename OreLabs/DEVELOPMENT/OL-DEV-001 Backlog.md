# OL-DEV-001 Backlog

## Purpose

The Backlog is the master list of all development work planned for OreLabs.

Every gameplay feature, technical task, content addition and improvement must exist here.

Tasks are never deleted.

Only their status changes.

---

# Task Status

Tasks may have one of the following statuses:

☐ Planned

◐ In Progress

☑ Completed

⏸ Deferred

❌ Cancelled

---

# Task Categories

Every backlog item belongs to one category.

## 🎯 Feature

A Feature introduces new gameplay or player-facing functionality.

Examples:

- New World
- New Pickaxe
- Pet Teams
- Egg System
- Forge
- Rebirth
- NPC

Features expand the game.

---

## 🔧 Technical

A Technical task improves the project without directly adding new gameplay.

Examples:

- Performance Optimization
- Save System Improvements
- Networking Refactor
- Memory Optimization
- Code Cleanup
- Architecture Refactor
- Debug Tools

Technical tasks improve stability, scalability and maintainability.

---

# Priority Levels

Every task should also have a priority.

Priority levels:

P0 — Critical

Blocks development or causes severe issues.

---

P1 — High

Required for the next milestone.

---

P2 — Normal

Important but not blocking.

---

P3 — Low

Quality-of-life improvements or future ideas.

---

# Task Format

Every backlog entry should follow the same structure.

Example:

[Feature] [P1] Mining System

Status:
☐ Planned

Description:
Implement the complete Mining gameplay loop.

Dependencies:

- Strength System
- Economy

Related Documents:

- OL-SYS-003
- OL-ARC-005

---

## Definition of Done

A task may only be marked as Completed when:

- implementation is finished;
- documentation is updated;
- testing is completed;
- no critical issues remain;
- the implementation follows the Architecture documents.

# Backlog Rules

- Never delete tasks.
- Every new feature must be added here.
- Tasks move between statuses instead of disappearing.
- Large tasks should be split into smaller tasks.
- The backlog is the single source of truth for development work.

---

# Success Criteria

The backlog is successful when:

- every planned feature exists in the backlog;
- developers always know what to work on next;
- completed work remains historically visible;
- development progress is transparent.

---

# Backlog Principle

> Nothing is forgotten.

> Everything is tracked.
