# Purpose

This document defines the long-term vision of OreLabs' first game.

It does not describe features or implementation details.

Instead, it defines the philosophy that every future system, mechanic and update must support.

---

# Vision

OreLabs is building a progression-driven Roblox experience where complexity grows naturally through player progression rather than through overwhelming design.

The objective is not to create the largest game.

The objective is to create a game that continuously rewards curiosity, mastery and long-term commitment.

Every new mechanic should expand the player's possibilities without replacing previously learned systems.

The game should always feel larger than it did yesterday while remaining understandable.

---

# Core Philosophy

Progression should never feel linear.

At every stage of the game the player should have multiple meaningful choices.

However, every choice should contribute to the same long-term journey.

The game grows like a tree with many branches connected to one trunk.

---

# Player Motivation

Players should constantly experience:

- curiosity
- discovery
- progression
- competition
- mastery
- collection
- optimization

The goal is not simply to earn larger numbers.

The goal is to unlock new possibilities.

---

# Progression Philosophy

Each progression system must answer a different player question.

Examples:

Strength answers:

"How powerful am I?"

Pickaxes answer:

"Where can I go?"

Backpacks answer:

"How long can I stay?"

Pets answer:

"How efficiently can I play?"

Rebirth answers:

"How far have I progressed?"

Future systems should continue this philosophy instead of duplicating existing mechanics.

---

# Long-Term Experience

The game should remain enjoyable after hundreds of hours.

Players should return because:

- new progression exists
- new strategies appear
- new builds become possible
- new goals emerge

Not because previous content became obsolete.

---

# Design Principles

Every update should satisfy at least one of the following:

- introduce meaningful choices;
- deepen an existing system;
- strengthen connections between systems;
- improve long-term progression;
- reward mastery rather than repetition.

Updates should never exist only to increase numbers.

---

# Success Definition

The project succeeds when players describe the game as:

"A game with incredible progression."

"A game where every mechanic has a purpose."

"A game that always gives me something meaningful to pursue."

Not simply:

"A game with many features."

---

# Vision Statement

OreLabs aims to build a game that becomes deeper over time without becoming more complicated.

Complexity must emerge through progression.

Never through confusion.
