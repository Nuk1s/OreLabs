# Purpose

This document defines the primary gameplay loop that drives the entire game.

Every mechanic introduced into the game must either strengthen this loop or naturally extend it.

If a mechanic does not contribute to the core loop, it should be reconsidered.

---

# Core Gameplay Principle

The player is always working toward unlocking new possibilities.

The gameplay loop is not based on repeating identical actions.

It is based on continuously expanding the player's available options.

Every completed cycle should unlock a new decision.

---

# Primary Gameplay Loop

The primary gameplay loop is built around continuous expansion of player capability.

The player follows this progression:

1. Mine available ores.
2. Collect resources.
3. Sell resources.
4. Earn Coins.
5. Train Strength.
6. Unlock stronger Pickaxes.
7. Access better ores and new mining areas.
8. Earn larger amounts of Coins and Gems.
9. Upgrade Backpack to extend mining sessions.
10. Improve Pets to specialize progression.
11. Increase overall efficiency.
12. Reach the current chapter's progression limit.
13. Perform Rebirth.
14. Unlock the next chapter of the game.
15. Repeat the loop with entirely new progression opportunities.

Each cycle should feel larger than the previous one.

The player is not repeating the same gameplay.

The player is expanding their available possibilities.

---

# Supporting Gameplay Loops

The main gameplay loop is supported by several secondary loops.

## Pet Loop

Collect Gems.

Open Eggs.

Discover better Pets.

Create stronger Pet combinations.

Increase efficiency.

Return to the main gameplay loop.

---

## Equipment Loop

Earn Coins.

Upgrade Pickaxe.

Unlock new ores.

Increase mining efficiency.

Return to the main gameplay loop.

---

## Backpack Loop

Earn Coins.

Upgrade Backpack.

Stay longer in mining areas.

Increase mining efficiency.

Return to the main gameplay loop.

---

## Progression Loop

Complete current progression stage.

Perform Rebirth.

Unlock the next chapter.

Discover entirely new content.

Return to the main gameplay loop.

---

# Active Gameplay

Active gameplay should provide the highest progression rate.

Players who actively participate should consistently progress faster than players who rely only on passive systems.

Exploration, mining, optimization and decision-making are the primary sources of progression.

---

# AFK Gameplay

AFK gameplay exists to complement active gameplay.

Its purpose is to maintain engagement during periods when the player cannot actively play.

AFK progression should never replace active progression.

The most valuable rewards should remain tied to active participation.

---

# Loop Expansion

As the player progresses, the gameplay loop expands rather than changes.

New systems should connect to the existing loop instead of replacing it.

Examples include:

- Forge
- Titles
- Crafting
- Future progression systems

Every future system should deepen the existing gameplay loop.

---

# Design Rules

Every gameplay loop must:

- create meaningful progression;
- encourage player decisions;
- interact with existing systems;
- remain understandable;
- support long-term replayability.

Loops that exist only to increase numbers should be avoided.

---

# Success Definition

The gameplay loop succeeds when players naturally alternate between multiple meaningful activities without feeling forced.

The player should never feel that there is only one correct thing to do.

Instead, the player should always have several valuable paths that contribute toward the same long-term progression.

---

# Gameplay Principle

A good gameplay loop does not ask the player to repeat actions.

It encourages the player to continuously discover better ways to play.
