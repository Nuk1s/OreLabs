# OL-DEV-004 Roadmap

## Purpose

The Roadmap defines the planned development order for OreLabs.

Unlike the Backlog, the Roadmap focuses only on the implementation sequence.

The objective is to deliver a playable game as early as possible while maintaining a stable and scalable architecture.

---

# Core Philosophy

Build the foundation first.

Deliver playable milestones early.

Expand the game incrementally.

Every phase should leave the project in a playable state.

---

# Roadmap Rules

- Follow phases in order whenever practical.
- Earlier phases take priority over later phases.
- New features should not delay foundation work.
- Every completed phase should produce a playable build.
- The Roadmap may evolve, but completed phases should remain unchanged.

---

# Relationship with the Backlog

The Backlog contains every planned task.

The Roadmap selects which tasks are implemented next.

The Roadmap is derived from the Backlog.

---

# Success Criteria

The Roadmap is successful when:

- development priorities remain clear;
- every phase produces measurable progress;
- the game remains playable after each milestone;
- the team always knows the next objective.

---

# Roadmap Principle

> Build a playable game first.

> Expand it second.

> Perfect it last.
